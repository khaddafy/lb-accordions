Kintassa Accordions - A Wordpress Plugin

Introduction

This plugin provides an easy, intuitive interface for creating accordions -- stacked panels with titles, which reveal the chosen panel when clicked or otherwise interacted with. Accordions are also known as toolboxes.

Features

  • Multiple Accordions
  • Multiple Panels per accordion
  • Horizontal and vertical accordions
  • Add any wordpress content to the accordion panels
  • Even nest accordions inside other accordions!
  • Plugin architecture for new/alternative rendering methods

Shortcodes and tags

Render a Accordion in post/page content using the kintassa_accordion shortcode, as follows:

[kintassa_accordion id="1"]

This will render the accordion from the admin that has the given id, according to the settings chosen in the admin area.

In php code, use the kintassa_accordion tag/function, as follows:

<?php echo kintassa_accordion(1); >

Styling

You'll undoubtedly want to style the accordions to suit your particular site. To do so, edit the stylesheets/kintassa_accordions.css file.

For more structural changes, edit the templates/applet_*_render.php files, where the * matches the short name for whichever display mode you have chosen, in the admin interface. For example, if you have chosen a vertical accordion, and would like to edit the generated HTML for this, modify the templates/applet_vert_accordion__render.php file. Accordion Javascript Settings

Some custom options are supported by the javascript accordion code itself. These options are provided by the template php files which write the javascript calls. For example, near the bottom of templates/applet_vert_accordion__render.php, you will see:

function () {

jQuery("#<?php echo $unique_id; ?>").msAccordion({
defaultid:0, vertical:true, start_collapsed:true
});
}

These default options -- defaultid, vertical, and start_collapsed -- can all be changed as needed. Options are as follows:

defaultid
The index of the default panel to show, counting from 0. For instance, to display the first panel by default, use defaultid:0. To display the second panel by default, use defaultid:1, and so on. Will only work if start_collapsed is false.
vertical
Whether to use vertical sliding instead of horizontal hiding when changing panels. true or false; defaults to false.
start_collapsed
Whether the panels should all be collapsed by default, when the accordion is first shown. true or false; defaults to false.

Custom Rendering Applets

To create a completely different rendering method, or to modify a rendering method without affecting existing methods (better for upgrading later), create a new applet in the applets folder, and a new template for it in the templates folder -- possibly by copying existing applets or templates. Then modify the new template and applet accordingly. In particular if copying, be sure to replace all mentions of the old applet name with your chosen new name.
